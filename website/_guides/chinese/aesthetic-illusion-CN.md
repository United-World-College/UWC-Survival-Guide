在有些社交媒体上的所谓社会精英叙事里，最优雅的莫过于“一边写代码、一边读莎士比亚”。就是在深夜的咖啡馆里，左手是满屏跳动的 CUDA 核函数，右手边恰到好处地摆着一本翻开的《哈姆雷特》的那种。这也恰好是某些精英教育流水线最笃信的配方。这种“通才”的人设极具诱惑力，因为它精准地切中了人们对阶层审美和浪漫叙事的向往。但如果你在大的实验室里待久了就会发现，这种叙事往往是极其危险的审美代偿。

*注：这里要谈的不是那些真心热爱两者的人，而是把博雅当 class signaling 的人。*

为什么很多人执着于莎士比亚呢？因为当一个人在处理系统死锁、优化 CUDA 核函数感到智力窒息时，把人文底蕴当作舒适区是最体面的退路。但这其实是一种错觉——真正的莎士比亚并不温情，他展现的是血淋淋的权力斗争和人性困境，其解构难度并不亚于任何复杂的分布式系统。所谓躲进人文，其实只是躲进了关于人文的幻象里。同样地，那些只堆砌代码逻辑、却拒绝思考工作意义的代码机器，也是另一种形态的逃避。

这种逻辑最令人沉溺的幻觉莫过于此："我代码写得不如 [ICPC](https://en.wikipedia.org/wiki/International_Collegiate_Programming_Contest) 战神，但我有更深厚的人文关怀。所以就算这些人现在在 ICPC 上屠榜了以后也得给我打工。"这种 class signaling 最隐秘的毒性就在于它将人文精神工具化，结果往往是既消解了技术的硬度，也抽空了人文的厚度。这不仅是对两者的双重损耗，更是一种智力上的自我粉饰。其实，拥抱平庸是每个人的权利。选择平平淡淡地度过四年、甚至是一辈子，或者承认自己只是将编程视为谋生手段，这是一种职业上的诚实，也是对自己的坦诚。

我其实一直觉得这种避难所心理，与在学术研究中有些人对灌水的盲目鄙视和对憋大招的病态执着极为相似。我曾经在网上看到过德州农工大学的一位教授对这种心理有过一段辛辣的评价：

> 有些人鄙视那些高产的灌水者，自己弄了几年，总以为憋大招能出一个[何恺明](https://en.wikipedia.org/wiki/Kaiming_He)或[李飞飞](https://en.wikipedia.org/wiki/Fei-Fei_Li)。结果东西磨出来后发现，还不如人家很多大实验室顺手灌水的成果。

这正是对学术虚荣的滑稽之处的讽刺。真正十年磨一剑的人，永远是那些敢于挑战大问题、在无人区硬碰硬的人，而不是那些躲在长期主义的幌子下自我感动的人。这对自诩“一边写代码、一边读莎士比亚”的人也是同理。当一个人试图用人文学科的格调来稀释技术焦虑时，他其实同时消解了两个领域的严肃性。最终，他既没能在代码的底层逻辑里获得力量，也没能在莎士比亚的悲剧里获得救赎。这种把博雅当 class signaling 的迷恋，最终只是让人在逻辑和审美的双重退缩中，把自己活成了一场滑稽的悲剧。

我一直都认为弄清楚一个文字的古代读音，其实与发现一颗小行星是同样重要的。这是因为任何领域的尊严，都只产生于解决复杂问题的阵痛之中。比起附庸风雅的挂靠，我更希望看到一个人在推敲文字的张力时，能像他在压榨代码的效能时一样，始终带着对底层逻辑的敬畏，去追寻那种纯粹的、不被学科边界所定义的真实。毕竟，真正的优雅从非逃避复杂的借口，而是征服复杂后的纯粹。
