在有些社交媒体上的所谓社会精英叙事里，最优雅的莫过于“一边写代码、一边读莎士比亚”。就是在深夜的咖啡馆里，左手是满屏跳动的 CUDA 核函数，右手边恰到好处地摆着一本翻开的《哈姆雷特》的那种。这也恰好是某些精英教育流水线最笃信的配方。这种“通才”的人设极具诱惑力，因为它精准地切中了人们对阶层审美和浪漫叙事的向往。但如果剥开这层优雅的滤镜，你会发现这种叙事往往只是一种极其危险的审美代偿。

*注：这里要谈的不是那些真心热爱两者的人，而是把博雅当 class signaling 的人。*

为什么很多人执着于莎士比亚呢？因为当一个人在处理系统死锁、优化 CUDA 核函数感到智力窒息时，把人文底蕴当作舒适区是最体面的退路。但这其实是一种错觉——真正的莎士比亚并不温情，他展现的是血淋淋的权力斗争和人性困境，其解构难度并不亚于任何复杂的分布式系统。所谓躲进人文，其实只是躲进了关于人文的幻象里。在这里，人文成了两拨人的共享赃物：有的技术人用它来逃避逻辑的挫败，有的文青用它来掩盖认知的空洞。前者觉得人文软，所以可以随手拿来垫脚；后者觉得人文虚，所以可以用来当遮羞布。他们共同构建了一个荒诞的共识：即人文是可以脱离严谨逻辑而独立存在的“氛围感”。但这恰恰是对人文最大的误解，因为没有逻辑支撑的情感永远只是廉价的煽情，没有深刻认知的文艺永远只是无病呻吟。实际上那些只堆砌代码逻辑、却拒绝思考工作意义的代码机器，与那些只会感叹风月、却拒绝理解世界运行底层逻辑的文青，本质上是同一种形态的逃避。

Class signaling 最令人沉溺的幻觉莫过于此："我代码写得不如 [ICPC](https://en.wikipedia.org/wiki/International_Collegiate_Programming_Contest) 战神，或者我根本就不屑于弄懂这些枯燥的逻辑，但我有更深厚的人文关怀。所以就算这些人现在在 ICPC 上屠榜了以后也得给我打工。"这种 class signaling 最隐秘的毒性就在于它将人文精神工具化，结果往往是既消解了技术的硬度，也抽空了人文的厚度。这不仅是对两者的双重损耗，更是一种智力上的自我粉饰。其实，拥抱平庸是每个人的权利。选择平平淡淡地度过四年、甚至是一辈子，或者承认自己只是将编程视为谋生手段，这是一种职业上的诚实，也是对自己的坦诚。事实上，承认自己既读不懂代码的逻辑，也读不进莎士比亚的十四行诗，甚至比这种以此代彼的优越感要高尚得多。

我其实一直觉得这种 class signaling 的心理，与在学术研究中有些人对灌水的盲目鄙视和对憋大招的病态执着极为相似。我曾经在网上看到过德州农工大学的一位教授对这种心理有过一段辛辣的评价：

> 有些人鄙视那些高产的灌水者，自己弄了几年，总以为憋大招能出一个[何恺明](https://en.wikipedia.org/wiki/Kaiming_He)或[李飞飞](https://en.wikipedia.org/wiki/Fei-Fei_Li)。结果东西磨出来后发现，还不如人家很多大实验室顺手灌水的成果。

这正是对学术虚荣的滑稽之处的讽刺。真正十年磨一剑的人，永远是那些敢于挑战大问题、在无人区硬碰硬的人，而不是那些躲在长期主义的幌子下自我感动的人。这对自诩“一边写代码、一边读莎士比亚”的人也是同理。当一个人试图用人文学科的格调来稀释技术焦虑时，他其实同时消解了两个领域的严肃性。最终，他既没能在代码的底层逻辑里获得力量，也没能在莎士比亚的悲剧里获得救赎。这种把博雅当 class signaling 的迷恋，最终只是让人在逻辑和审美的双重退缩中，把自己活成了一场滑稽的悲剧。

我一直都认为弄清楚一个文字的古代读音，其实与发现一颗小行星是同样重要的。这是因为任何领域的尊严，都只产生于解决复杂问题的阵痛之中。比起附庸风雅的挂靠，我更希望看到一个人在推敲文字的张力时，能像他在压榨代码的效能时一样，始终带着对底层逻辑的敬畏，去追寻那种纯粹的、不被学科边界所定义的真实。毕竟，真正的优雅从非逃避复杂的借口，而是征服复杂后的纯粹。

## 附录：一个关于诚实的自检清单

在 UWC 这种期待值极高的环境下，我们很难不被某种“完美精英”的叙事裹挟。如果你也曾怀疑过自己是否在进行某种 Class Signaling，不妨在深夜独处时，试着温柔地问自己几个问题：

**关于观众：**
如果在你的生活里，没有任何人会知道你在读什么、在学什么，甚至没有人会因为你的这种“博雅”而给你加分，你是否依然会被那些文字或逻辑本身所打动？如果那种纯粹的喜悦消失了，或许是因为你背负了太沉重的、演给别人看的剧本。

**关于优越感：**
你是否在潜意识里，把这种跨学科的身份当成了一种防御机制？比如，当你在某个领域遇到瓶颈时，会通过“我还有另一种深度”来消解当下的挫败。真正的底蕴不应该是用来俯视他人的台阶，而应该是支撑你走过低谷的底气。

**关于避难所：**
你是在消费某种符号，还是在体验它背后的精神？无论是躲进书本还是躲进代码，承认自己累了、需要一个喘息的空间应该被鼓励。我们需要区分的是：你是在寻找一个让自己活下去的避难所，还是在修筑一座用来彰显不同的护城河？

**关于笨拙的诚实：**
你是否允许自己在跨界时表现得“不优雅”？真正的跨学科探索在初期往往都是狼狈的。如果你因为害怕破坏“文理双修”的人设而不敢承认自己在某个领域其实只是个笨拙的初学者，那么这种人设本身就是你成长的枷锁。

**写在最后：**

承认自己偶尔需要“人设”并不羞耻，这甚至是社交生活的一种润滑剂。但请警惕，别让这些信号成为了你与真实世界、真实自我之间的围墙。写这篇文章并不是为了批判那些在写代码间隙翻开书页的人。恰恰相反，在这个人人都想成为完美精英叙事下的主角的时代，我更想致敬的是那些敢于承认自己没读懂莎士比亚的勇气，或坦然面对代码逻辑平庸的诚实。因为我们永远不需要用另一种高度来掩盖当下的狼狈。

最好的优雅，是当你终于不需要向任何人证明自己的时候，依然对这个世界的运行——无论是那行难搞的代码，还是那段晦涩的诗句——保有那一丝纯粹的好奇与敬畏。
